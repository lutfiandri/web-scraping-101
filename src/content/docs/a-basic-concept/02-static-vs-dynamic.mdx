Pernah kepikiran gak, kenapa scraping Tokopedia rasanya jauh lebih susah daripada scraping Wikipedia? Padahal sama-sama website, kan?

Jawabannya ada di perbedaan mendasar cara mereka menyajikan data. Anggap aja kamu lagi mau beli buku di dua toko yang beda.

**Toko Pertama (Website Statis):** Rak-raknya udah penuh buku, lengkap dengan label harga. Kamu tinggal masuk, ambil buku yang kamu mau, bayar, dan pulang. Semuanya udah siap sedia.

**Toko Kedua (Website Dinamis):** Tokonya keliatan kosong. Kamu harus panggil pelayan dulu, bilang mau buku apa, terus pelayan itu lari ke gudang, nyari bukunya, baru deh dikasih ke kamu. Kadang, harganya bisa beda sama pas kamu liat di katalog.

Di dunia web scraping, kedua "toko" ini butuh cara penaklukan yang sama sekali berbeda. Salah strategi, bisa-bisa scraper kamu malah bengong atau datanya zonk.

## Website Statis

Website statis itu ibaratnya kayak koran atau brosur. Isinya udah dicetak dari sananya, gak bakal berubah mau kamu lihat dari HP atau laptop, mau kamu buka sekarang atau besok.

### Gimana Cara Kerjanya?

![static website flow](/a-02-static-web.excalidraw.svg)

Prosesnya super simpel:

1.  **Kamu (Browser):** "Server, minta halaman `web.com/about` dong!"
2.  **Server:** "Oke, nih ambil filenya!" (Langsung ngasih file HTML utuh)
3.  **Kamu (Browser):** Tinggal nampilin isinya. Selesai!

Gampang, kan? Semua informasi yang kamu lihat di layar itu udah ada di dalam file HTML yang kamu terima pertama kali.

### Ciri-ciri Website Statis

Kalau kamu ragu, coba trik detektif ini:

> Klik kanan di halaman web > `View Page Source`. Terus `Ctrl+F` dan cari sepotong teks yang kamu lihat di halaman itu.

Kalau ketemu, **99% pasti itu website statis**. Selamat! Misi scraping kamu bakal gampang.

![Contoh View Source di web statis](/a-02-static-view-page-source.png)

Kenapa datanya ada? Ya karena semua kontennya memang udah "dipanggang" jadi satu di dalam file HTML-nya.

## Website Dinamis

Nah, ini dia biang keroknya. Website dinamis itu kayak feed Instagram atau halaman produk di e-commerce. Kontennya bisa beda-beda tiap orang, tergantung siapa kamu, kapan kamu buka, atau bahkan seberapa jauh kamu scroll.

Kontennya gak langsung ada di file HTML awal. Halamannya itu kayak kerangka kosong yang "dirakit" isinya belakangan pakai JavaScript.

### Gimana Cara Kerjanya?

![dynamic website flow](/a-02-dynamic-web.excalidraw.svg)

Prosesnya jadi ga sesederhana tadi:

1.  **Kamu (Browser):** "Server, minta halaman `web.com/feeds`!"
2.  **Server:** "Nih kerangkanya dulu (HTML + JavaScript), datanya nyusul."
3.  **Browser:** Nampilin kerangka kosong.
4.  **JavaScript di Browser:** "Woi Server, datanya mana buat feeds?" (Ini namanya request ke API)
5.  **Server:** "Oke, nih data feeds-nya dalam format JSON."
6.  **JavaScript di Browser:** "Sip, data diterima!" (Lalu data JSON itu "disuntikkin" ke kerangka HTML tadi).

Tada! Kamu udah bisa lihat feeds-nya sekarang.

<Aside>
  Intinya, ada **dua tahap** request. Babak pertama minta kerangka, tahap kedua
  minta isinya.
</Aside>

### Ciri-ciri Website Dinamis

Uji lagi dengan trik detektif yang sama:

> Klik kanan > `View Page Source` > `Ctrl+F` > cari harga atau judul produk.

Kalau gak ketemu, **99% pasti itu website dinamis**.

![Contoh View Source di web dinamis](/a-02-dynamic-view-page-source.png)

Kenapa datanya gak ada? Karena data aslinya baru diminta sama JavaScript _setelah_ kerangka HTML-nya nyampe di browser kamu.

## Perang Scraper: Statis vs Dinamis

"Terus ngaruhnya apa buat aku?"

Ngaruh banget! Ini nentuin senjata apa yang harus kamu bawa ke medan perang.

**Melawan Website Statis:**
Cukup bawa `requests` di Python atau `curl` di terminal. Satu tembakan (request) langsung dapet semua datanya. Cepat, ringan, hemat resource. Ibaratnya cuma nembak pake pistol.

**Melawan Website Dinamis:**
Kamu butuh senjata berat: **browser asli yang bisa dikontrol kode**, kayak [Playwright] atau Selenium. Kenapa? Karena scraper kamu harus sabar nungguin semua drama request data oleh JavaScript selesai.

Konsekuensinya?

- **Lambat:** Nunggu browser render itu butuh waktu.
- **Boros RAM:** Buka browser itu makan memori, apalagi kalau puluhan sekaligus.
- **Lebih kompleks:** Kode kamu harus bisa menangani timing, klik, dan scroll.

<Aside type="tip">
  Kamu bisa aja pake Playwright buat scraping web statis. Tapi itu kayak nembak
  nyamuk pake bazoka. Bisa, tapi boros dan gak efisien.
</Aside>

Jadi, identifikasi dulu apakah targetmu itu website statis atau dinamis biar scraping-mu lebih optimal.
